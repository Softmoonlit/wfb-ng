## 区域 4: 应用层集成验证方法

### 验证方法

| Option | Description | Selected |
|--------|-------------|----------|
| 网络抓包分析 | 使用 tcpdump/wireshark 捕获网络流量，验证数据包格式和时序 | ✓ |
| 应用层日志分析 | 分析 UFTP 和 wfb-ng 的日志输出，验证交互正确 | ✓ |
| 系统调用追踪 | 使用 strace/ltrace 跟踪系统调用，验证通信路径 | |
| 内核态监控 | 监控 TUN 设备队列、网卡状态、内核统计 | |

**User's choice:** 网络抓包分析 + 应用层日志分析
**Notes:** 结合网络层和应用层的可见性，全面验证

---

### 路由配置

| Option | Description | Selected |
|--------|-------------|----------|
| 全局路由（所有流量经过 TUN） | 最简单，所有 UDP 流量都经过 TUN，但可能影响其他应用 | |
| 特定 IP 段路由 | 精确控制，仅目标 IP 匹配时经过 TUN，不影响其他应用 | ✓ |
| 应用层绑定 TUN 接口 | 让 UFTP 直接绑定 tun0 接口，修改 UFTP 源码（不推荐，破坏零感知） | |

**User's choice:** 特定 IP 段路由
**Notes:** 精确控制，不影响其他应用，保持应用层零感知

**User clarification:** 用户提出疑问"UFTP 作为应用层不是和 TUN 交互数据吗？"，澄清了架构：
- UFTP 通过标准网络接口（eth0）发送 UDP 包
- TUN 是透明拦截层，通过路由配置将特定 IP 段的流量导向 TUN 设备
- UFTP 不直接与 TUN 交互，保持零感知

---

### 验证内容

| Option | Description | Selected |
|--------|-------------|----------|
| TUN 设备拦截验证 | 验证 UFTP 数据包经过 TUN 设备，且被 wfb_core 正确处理 | ✓ |
| 控制帧机制验证 | 验证 Radiotap 模板切换、三连发、保护间隔等机制工作正常 | ✓ |
| 数据完整性验证 | 验证应用层收到的数据与发送的一致，无丢失或损坏 | |
| 零感知验证 | 验证 UFTP 无需修改源码，无需感知底层调度 | ✓ |

**User's choice:** TUN 设备拦截验证 + 控制帧机制验证 + 零感知验证
**Notes:** 核心验证项，确保集成正确

---
