实现 AQ/SQ 队列管理器核心逻辑：

1. `init_node(node_id)` 实现:
   - 加锁
   - 将节点加入 SQ 队尾
   - 更新 in_sq_set

2. `migrate_to_aq(node_id)` 实现（per RESEARCH.md Pattern 2）:
   - 加锁
   - 如果已在 AQ，直接返回
   - 从 SQ 移除（使用 erase + remove）
   - 从 in_sq_set 移除
   - 加入 AQ 队尾
   - 更新 in_aq_set

3. `migrate_to_sq(node_id)` 实现:
   - 加锁
   - 如果已在 SQ，直接返回
   - 从 AQ 移除
   - 从 in_aq_set 移除
   - 加入 SQ 队尾
   - 更新 in_sq_set

4. `get_next_node(interleave_ratio)` 实现（per SCHED-10）:
   - 加锁
   - 场景 1: AQ 为空且 SQ 为空 → 返回 {INVALID_NODE, false}
   - 场景 2: AQ 为空但 SQ 不空 → 空闲巡逻模式，从 SQ 轮询取节点，返回 {node, true}
   - 场景 3: 混合交织调度:
     - 维护静态计数器 aq_served
     - 每服务 interleave_ratio 次 AQ，穿插 1 次 SQ
     - 若 SQ 为空或 aq_served < interleave_ratio，服务 AQ
     - 否则服务 SQ 并重置计数器
   - 轮询实现: 从队首取节点，放回队尾

interleave_ratio 默认值为 4（介于 3-5 之间，per SCHED-10）。

关键约束:
- 所有操作必须在锁保护下进行
- 节点迁移时必须同时更新队列和对应的 set
- 轮询操作：取出队首，放回队尾
