# Plan 05-05: 启动脚本更新

## Objective

更新 server_start.sh 和 client_start.sh，使用 wfb_core 替代原有的多进程方案（wfb_tx/wfb_rx/wfb_tun），保持向后兼容性。

## Background

当前启动脚本仍然启动多个独立进程。需要将其改为启动单个 wfb_core 进程，同时保留环境准备逻辑（Monitor 模式、TUN 创建等）。

### 1. 更新 server_start.sh

保留现有的环境准备逻辑（Monitor 模式、权限检查），将进程启动段改为：

```bash
# === 启动 wfb_core 单进程 ===
log_info "启动 wfb_core 服务端..."

# 构建 wfb_core（如果需要）
if [ ! -f "$WFB_CORE_BIN" ]; then
    log_warn "wfb_core 未找到，尝试编译..."
    make clean && make
fi

# 启动 wfb_core
$WFB_CORE_BIN \
    --mode server \
    -i "$WIFI_INTERFACE" \
    -c "$CHANNEL" \
    -m "$MCS" \
    --bw "$BANDWIDTH" \
    --tun "$TUN_NAME" \
    $VERBOSE_FLAG &

WFB_PID=$!
log_success "wfb_core 已启动 (PID: $WFB_PID)"

# 保存 PID 到文件
echo $WFB_PID > "$PID_FILE"

# 等待进程退出
wait $WFB_PID
EXIT_CODE=$?

log_info "wfb_core 已退出 (退出码: $EXIT_CODE)"
```

### 2. 更新 client_start.sh

```bash
# === 启动 wfb_core 单进程 ===

$WFB_CORE_BIN \

    -i "$WIFI_INTERFACE" \
    -c "$CHANNEL" \
    -m "$MCS" \
    --bw "$BANDWIDTH" \

log_success "wfb_core 已启动 (PID: $WFB_PID, 节点 ID: $NODE_ID)"

echo $WFB_PID > "$PID_FILE"

wait $WFB_PID
EXIT_CODE=$?

log_info "wfb_core 已退出 (退出码: $EXIT_CODE)"
```

### 3. 清理函数更新

```bash
# 优雅退出函数
cleanup() {
    log_info "收到退出信号，清理资源..."

    if [ -n "$WFB_PID" ] && kill -0 $WFB_PID 2>/dev/null; then
        log_info "发送 SIGTERM 到 wfb_core (PID: $WFB_PID)..."
        kill -TERM $WFB_PID
        wait $WFB_PID 2>/dev/null
    fi

    # 删除 TUN 设备
    if ip link show "$TUN_NAME" &>/dev/null; then
        ip link delete "$TUN_NAME" 2>/dev/null
    fi

    log_success "清理完成"
    exit 0
}

trap cleanup SIGINT SIGTERM
```

### 5. 创建部署文档

创建 `docs/deployment.md`，记录：

```markdown
# 部署指南

## 单进程模式（推荐）

### 服务端启动
\`\`\`bash
sudo ./scripts/server_start.sh -i wlan0 -c 6 -m 0
\`\`\`

### 客户端启动
\`\`\`bash
sudo ./scripts/client_start.sh -i wlan0 -c 6 -m 0 --node-id 1
\`\`\`

## 参数说明

- \`--mode\`: server 或 client
- \`-i\`: WiFi 网卡接口
- \`-c\`: 信道号
- \`-m\`: MCS 调制编码方案（0-8）
- \`--bw\`: 频宽（20 或 40 MHz）
- \`--tun\`: TUN 设备名称
- \`--node-id\`: 节点 ID（客户端专用，1-255）

## 权限要求

必须以 Root 权限运行（SCHED_FIFO 实时调度需要）：
\`\`\`bash
sudo ./scripts/server_start.sh ...
\`\`\`

## 性能优化

- 设置 CPU 亲和性（避免核心切换）
- 绑定实时线程到专用核心
- 关闭 CPU 频率调节（性能模式）

## 故障排查

### 实时调度失败
\`\`\`bash
# 检查权限
whoami  # 应为 root

# 检查内核支持
cat /proc/sys/kernel/sched_rt_runtime_us
\`\`\`

### TUN 设备创建失败
\`\`\`bash
# 检查 TUN 模块
lsmod | grep tun

# 手动加载
modprobe tun
\`\`\`
```
