## 架构决策

### 1. 默认单进程模式
**决策**: 将单进程模式设为默认启动方式  
**理由**: 符合项目核心架构要求，消除多进程通信延迟和丢包风险  
**实现**: 去除多进程启动代码，专注单进程实现

### 2. 向后兼容设计
**决策**: 保留 `--legacy` 参数但不实现完整逻辑  
**理由**: 保持命令行接口兼容性，但不鼓励使用旧模式  
**实现**: 显示警告信息并提示使用单进程模式

### 3. 统一信号处理
**决策**: 所有脚本使用相同的清理函数  
**理由**: 确保资源正确释放，避免僵尸进程和残留设备  
**实现**: `cleanup()` 函数处理进程终止、TUN 删除、PID 文件清理

### 4. 动态编译支持
**决策**: 如果 wfb_core 未找到，尝试自动编译  
**理由**: 简化部署流程，减少手动步骤  
**实现**: 检查可执行文件，不存在时运行 `make clean && make`

## 技术细节

### 参数解析改进
```bash
# 统一参数格式
-i wlan0           # WiFi 接口
-c 6              # 信道
-m 0              # MCS 调制方案
--bw 20           # 频宽 (MHz)
--tun wfb0        # TUN 设备名
--node-id 1       # 客户端节点 ID
--fec-n 12        # FEC 总块数
--fec-k 8         # FEC 数据块数
```

### 信号处理机制
```bash
# 注册清理函数
trap cleanup SIGINT SIGTERM

# 清理函数逻辑
cleanup() {
    # 1. 终止 wfb_core 进程
    # 2. 删除 TUN 设备
    # 3. 删除 PID 文件
    # 4. 退出脚本
}
```

### 日志管理
- 日志文件: `/var/log/wfb_core_{server,client}.log`
- 日志格式: `[LEVEL] 时间戳 消息`
- 输出方式: `tee` 同时输出到控制台和文件
- 日志轮转: 依赖系统 logrotate（未来增强）

### 权限管理
- **Root 要求**: 脚本开始检查 `EUID`，确保实时调度权限
- **参数验证**: 所有输入参数验证范围和格式

### 资源清理

- **设备删除**: TUN 设备使用后自动删除
- **文件清理**: PID 文件在退出时删除

## 性能影响

### 启动时间优化
- **冷启动**: 如果 wfb_core 未编译，自动编译增加约 30-60 秒
- **热启动**: 直接启动约 1-2 秒
- **Monitor 模式设置**: 网卡配置约 2-3 秒

### 资源占用
- **进程数**: 从 3 个进程减少到 1 个进程
- **内存**: 单进程内共享内存，减少重复缓冲
- **CPU**: 线程间通信开销降低

## 已知限制

### 向后兼容性
- `--legacy` 参数仅显示警告，未实现完整多进程启动
- 旧的环境变量（如 `WLAN_IFACE`, `MCS_INDEX`）不再支持
- UDP 端口配置参数已移除

### 平台依赖
- 依赖 `iw` 和 `ip` 命令设置 Monitor 模式
- 需要 Linux 内核 TUN 支持
- 实时调度需要内核配置支持

### 日志管理
- 日志文件不会自动轮转
- 日志级别控制有限（仅 verbose 模式）
- 没有结构化日志输出
