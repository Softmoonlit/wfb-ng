## 核心实现

### 1. RAII 资源管理扩展 (resource_guard.h)
- **添加 `make_zfex_handle()`**: 创建 zfex_code 智能指针的辅助函数
- **添加 `make_fd_handle()`**: 创建文件描述符智能指针的辅助函数
- **前向声明支持**: 添加必要的头文件包含和前向声明

### 2. TUN 读取线程主循环 (tun_reader.cpp)
- **动态水位线反压**: 基于 MCS 速率映射表计算动态水位线，队列满时停止读取
- **FEC 配置验证**: 验证 FEC 参数有效性（N, K 配置）
- **错误恢复机制**: 连续错误检测与退避策略
- **RAII 资源管理**: TUN 文件描述符使用 FdHandle 自动管理
- **条件变量通知**: 数据入队后通知发射线程

### 3. 关键设计决策
1. **简化 FEC 编码实现**: 保留配置验证接口，实际 FEC 编码逻辑待后续计划实现
2. **动态水位线计算**: 基于物理速率和 MTU 动态计算，避免硬编码
3. **普通优先级线程**: TUN 读取线程保留普通优先级，避免 FEC 编码 CPU 密集型操作导致系统死机

## 技术架构

### 数据流路径
```
TUN 设备 → 非阻塞读取 → 动态水位线检查 → 数据入队 → 通知发射线程
       ↓              ↓
    RAII 管理     反压机制 (1ms 退避)
```

### 资源管理
- **TUN 文件描述符**: 使用 `FdHandle` RAII 包装，确保安全释放
- **FEC 编码器**: 预留 `ZfexCodeHandle` 接口，实际管理待后续实现
- **内存缓冲区**: 使用 `std::vector<uint8_t>` 自动管理生命周期
