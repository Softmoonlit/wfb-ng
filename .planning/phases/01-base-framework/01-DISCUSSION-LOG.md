## 序列号生成机制（CTRL-03）

### 问题 1：序列号生成器的线程安全机制
| 选项 | 描述 | 已选择 |
|------|------|--------|
| 原子变量（std::atomic） | 使用 std::atomic<uint32_t>，无锁并发，性能最优 | ✓ |
| 互斥锁（std::mutex） | 使用 std::mutex 保护的普通 uint32_t，与现有 packet_queue.h 模式一致 | |
| GCC 内置函数 | 使用 __sync_fetch_and_add() 内置函数，无锁但非标准 C++ | |

**用户选择：** 原子变量（std::atomic）

**决策：** D-01 - 使用 `std::atomic<uint32_t>` 实现全局序列号生成器

---

### 问题 2：序列号初始化策略
| 选项 | 描述 | 已选择 |
|------|------|--------|
| 从 0 开始 | 从 0 开始，简单直观，重启后序列号重置 | ✓ |
| 基于时间戳 | 使用当前时间戳（秒或毫秒）作为初始值 | |
| 随机初始值 | 使用随机数生成初始值 | |

**用户选择：** 从 0 开始

**决策：** D-02 - 序列号从 0 开始初始化

**备注：** 系统重启后所有旧包已失效，无需复杂初始化
