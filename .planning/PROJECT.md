# 联邦学习无线空口底层传输架构

## What This Is

基于 wfb-ng 改造的具有深缓冲区与动态时间配额轮询（Token Passing）的无碰撞传输架构，彻底解决联邦学习在半双工无线空口中的射频碰撞与拥塞问题。系统严格分为三层：应用层（UFTP 零感知微超发）、操作系统层（Linux TUN 极小水池防抖动）、MAC/物理层（魔改 wfb-ng，无碰撞调度层）。合并 TUN 读取与 TX 发射进程，使用动态水位线产生基于丢包的物理背压；服务端作为中心调度器，采用"呼吸"周期广播大模型，空闲和高负载时分别应用交织探询与三态弹性扩窗调度。

## Core Value

基于深缓冲区与动态时间配额轮调度的无碰撞传输架构，彻底消灭射频碰撞并逼近物理极限带宽利用率，确保联邦学习 40MB 模型参数稳定传输。

## Context

基于联邦学习（FL）与高延迟/半双工无线空口（wfb-ng）深度定制。物理层痛点：wfb-ng 基于 WiFi Monitor 模式，没有 CSMA/CA 冲突避免机制，多节点同时发送会导致灾难性的物理层射频碰撞。业务层痛点：联邦学习具有非对称性（1个服务器对10个客户端）、突发性（40MB模型/梯度文件突发传输）和异步性（各客户端本地训练时间随机）。核心解法：废弃不可控的微观并发，采用"中心强管控的时间配额轮询（Token Passing）"机制，结合 Linux 操作系统的"深水池缓冲队列"，将碰撞问题在 MAC 层彻底消灭，并对上层业务（UFTP）保持绝对透明。

## Constraints

- **技术栈**: C/C++, Linux (TUN 设备, 实时调度 SCHED_FIFO, poll/epoll), libpcap/Raw Sockets (WiFi Monitor Mode) — 必须使用纯 C/C++ 编写调度器以消除运行时抖动毛刺
- **物理层**: 基于 wfb-ng 架构改造，保留现有 Monitor 模式和 FEC 机制（仅控制帧跳过）
- **实时性**: 调度器主线程必须提权至 SCHED_FIFO 最高实时优先级，使用 timerfd 配合极短时间空转自旋，确保微秒级绝对精度
- **无线环境**: 半双工 WiFi 空口，无 CSMA/CA 冲突避免，必须由中心调度器控制发射权
- **数据规模**: 单次传输 40MB 模型文件，10 节点并发
- **应用层兼容性**: 对 UFTP 零感知，继续使用标准 UDP/IP 协议
