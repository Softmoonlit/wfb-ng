## 脆弱区域

**跨平台开发风险:**
- 文件: `Cargo.toml`, `.gitignore`, 以及所有带 CRLF 换行符的源码文件
- 脆弱原因: 项目在 Windows (当前工作目录) 编写，但最终目标（Monitor 模式网卡、TUN/TAP、libpcap 等）高度依赖 Linux 内核特性。如果存在换行符问题(CRLF vs LF)或者文件执行权限丢失(如 shell 脚本)，会导致 Linux 侧编译或运行失败
- 安全修改: 在 `.gitattributes` 中强制 `* text=auto eol=lf`
- 测试覆盖: 目前缺乏基于真实 Linux 和 Monitor 模式硬件的自动化测试
